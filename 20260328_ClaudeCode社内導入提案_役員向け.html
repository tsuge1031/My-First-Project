01

Claude Code とは何か

開発元・位置づけ

米Anthropic社が提供するターミナルベースのAIコーディングアシスタント。ChatGPTと異なり開発者のローカル環境で直接動作し、コードの読み書き・テスト実行・バグ修正を自律的に行う。

対応範囲

ファイルの読み書き・Git操作・テスト実行・コードレビュー・ドキュメント生成など開発工程全般をカバー。自然言語での指示で操作でき、エンジニアの補助ツールとして機能する。

「AIがコードを書き、人間が判断する」新しい開発スタイル ChatGPTのような「質問→回答」ではなく、AIが開発環境に直接接続してタスクを実行するエージェント型ツール
02

メリット・デメリット

✅ メリット（導入効果）

▲
開発生産性の向上（最大30〜50%）

定型的なコード生成・テスト作成・バグ調査の時間を大幅削減。エンジニアが設計・レビューなど高付加価値業務に集中できる。

▲
品質・一貫性の向上

コーディング規約の自動適用、テストコード自動生成により人的ミスを削減。ドキュメント生成も自動化でき、保守性が向上。

▲
人材育成・スキル平準化

ジュニアエンジニアがシニアレベルの知識を補完できる。技術の属人化リスクを低減し、チーム全体のスキル底上げに寄与。

▲
競争力・受注力の強化

工数削減による価格競争力向上。AI活用による提案力・技術力のアピールで新規案件獲得につながる。

▲
残業削減・働き方改革

繁忙期のコーディング・デバッグ作業をAIが肩代わりし、過重労働の抑制・エンジニア離職防止に効果的。

⚠️ デメリット・留意点

!
顧客情報の外部送信リスク

デフォルト設定では入力内容がAnthropicのサーバーに送信される。顧客情報・機密コードを誤って含めた場合のリスクがある。→ 対策あり（次頁参照）

!
AIの出力品質のバラつき

生成されたコードが常に正しいとは限らない。エンジニアによるレビュー・検証が必須であり、過信による品質事故のリスクがある。

!
ランニングコストの発生

APIの利用量に応じた従量課金が発生。利用が増えるほどコストが上昇するため、使用ルールの策定が必要。

!
社員のスキル依存度の変化

AIへの過度な依存により、エンジニア自身のスキル形成が遅れる懸念がある。適切な利用ガイドラインの整備が必要。

!
導入・管理コストの発生

PoC設計・ガイドライン策定・社員教育・利用監視の体制整備に初期工数が必要。IT部門への負担が一時的に増加。

03

セキュリティ対策（SIerとして最重要）

🔒

学習利用なし

Team / Enterprise / APIプラン契約下では、送信したコードや指示文がAIモデルの学習に一切使用されないことが契約で保証されている

🗄️

ゼロデータ保持（ZDR）

Enterpriseプランで申請可能。推論後にプロンプトとレスポンスをサーバーから即時削除。金融・医療・官公庁案件にも対応できる最高レベルの保護

🛡️

国際認証取得済み

SOC 2 Type II・ISO 27001・ISO/IEC 42001（AI管理）・HIPAA BAA対応。第三者機関による審査済みのセキュリティ体制が確立されている

☁️

自社クラウド内完結

AWS Bedrock / Google Vertex AI経由で利用すると、データがAnthropicのサーバーに送られず自社クラウド環境内で完結。既存のセキュリティポリシーをそのまま適用できる

📋

監査ログ完備

AWS CloudTrailによる自動記録。誰がいつ何を入力したかを追跡可能。セキュリティインシデント発生時の調査・証跡として活用できる

⚙️

権限管理・ポリシー強制

managed-settings.jsonにより、使用できる機能・アクセス可能なファイルを組織ポリシーとして強制適用。MDMと連携し全端末へ一括配布も可能

リスクと対応策

リスク 影響度 対応策
顧客情報・機密コードの外部送信
エンジニアが誤って顧客DBスキーマや個人情報を含むコードをAIに送信		 高 AWS Bedrock経由でデータを自社AWS内に限定。managed-settings.jsonで.envファイル・認証情報へのアクセスを禁止。利用禁止コンテンツのガイドライン策定と研修
AI生成コードの品質問題
AIが生成したコードに誤りがあり、そのまま本番に適用されてしまう		 中 AI生成コードは必ずエンジニアがレビュー・テストしてから採用するルールを義務化。自動テストの整備を並行推進。承認フローを既存の開発プロセスに組み込む
シャドーAI（無許可利用）の発生
社員が個人契約のAIツールを無断使用し、顧客情報が流出するリスク		 高 公式ツールとして社内整備することで「公認の抜け道」を提供し、シャドーAIの動機を排除。セキュリティポリシーで無断利用を禁止・周知徹底
コスト超過
API利用量が想定を超え、月次コストが膨らむ		 中 AWS Cost Explorerでリアルタイム監視。部門ごとの利用上限を設定。月次レビューで使用状況を確認・調整
エンジニアの技術力低下
AIへの過依存により自力でコードを書く能力が失われる		 低 利用ガイドラインで「AIは補助ツール、最終判断は人間」を明記。定期的なコードレビュー文化を維持。新人研修では基礎スキルを従来通り育成

🎯 経営判断のポイント

1

今すぐ導入すべき理由競合SIerのAI活用は急速に進んでいる。導入の遅れは価格競争力・提案力の差として顕在化し、受注機会の損失につながる。

3

シャドーAIを放置することの方がリスク既に多くの社員が個人でAIツールを使っている可能性が高い。公式導入してガバナンスを効かせる方が、情報漏洩リスクを確実に低減できる。

4

推奨アクションまず5〜10名でPoC承認を。AWS Bedrock経由のセキュアな環境で2ヶ月間検証し、定量的な効果確認後に全社展開を判断する。
