# 静的サイトホスティングサービス比較

> 調査日：2026年4月3日　｜　対象：HTMLファイルをワンコマンド〜手軽に公開できるサービス

---

## 概要

Surgeと同様に、静的HTMLを簡単に公開できるサービスの比較。

---

## サービス一覧

### 無料・手軽系（Surgeに近い）

| サービス | 特徴 | CLIデプロイコマンド |
|---------|------|-------------------|
| **[Surge](https://surge.sh)** | CLIワンコマンドで公開。ドメイン名を自由に指定できる | `npx surge . --domain xxx.surge.sh` |
| **[Netlify Drop](https://app.netlify.com/drop)** | ブラウザにドラッグ＆ドロップするだけで公開。CLIも使える | `npx netlify deploy` |
| **[Vercel](https://vercel.com)** | Next.js作者が作成。CLIワンコマンドで公開 | `npx vercel` |
| **[GitHub Pages](https://pages.github.com)** | GitHubリポジトリから自動公開。無料・独自ドメイン対応 | `git push` |
| **[Cloudflare Pages](https://pages.cloudflare.com)** | CDN世界最大手。高速・無料プランが充実 | `npx wrangler pages deploy` |

### 少し設定が必要な上位サービス

| サービス | 特徴 |
|---------|------|
| **[Firebase Hosting](https://firebase.google.com/products/hosting)** | Google製。`firebase deploy` 一発。SSL自動 |
| **[Render](https://render.com)** | 静的サイト＋サーバーも動く。GitHubと連携して自動デプロイ |

---

## 比較まとめ

### 手軽さ（CLIワンコマンド）
```
Surge ≒ Vercel > Netlify > Cloudflare Pages > GitHub Pages
```

### 速度・CDN
```
Cloudflare Pages > Vercel ≒ Netlify > Surge > GitHub Pages
```

### 無料枠の充実度
```
Cloudflare Pages > Netlify > Vercel > GitHub Pages > Surge
```

### カスタムドメイン対応
```
全サービスで対応
```

---

## 用途別おすすめ

| 用途 | おすすめ | 理由 |
|------|---------|------|
| 社内提案・図解資料の一時共有 | **Surge** | ドメイン名を内容に合わせて指定できる。GitHubリポジトリ不要。ワンコマンド完結 |
| 手軽に試したい（ブラウザのみ） | **Netlify Drop** | ファイルをドラッグするだけ。アカウント不要 |
| 継続的に運用するサイト | **Cloudflare Pages** | 無料枠が最も充実。CDN高速 |
| GitHubで管理しているプロジェクト | **GitHub Pages** | pushするだけで自動公開 |

---

## 結論

社内提案・図解資料の共有用途には **Surge が最適**。  
次に試すとしたら **Netlify Drop**（ブラウザでドラッグするだけ）が手軽でおすすめ。
