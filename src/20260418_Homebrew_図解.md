# Homebrew 図解資料（原本）

- **調査日**: 2026.04.18
- **対象**: macOS / Linux 向けパッケージマネージャ Homebrew（brew.sh）

---

## 概要（WHAT）

Homebrew は、**macOS および Linux** で動作するオープンソースのパッケージマネージャ。OS に標準で入っていない開発ツールや CLI アプリを、**コマンド一行で取得・更新**しやすくする。公式のキャッチコピーは「The Missing Package Manager for macOS (or Linux)」。

---

## 背景・目的（WHY）

- アプリごとに公式サイトから dmg を落として…という手作業を減らす。
- 依存ライブラリ（OpenSSL など）もまとめて解決しやすい。
- チームや CI で「同じツールチェーン」を再現しやすい（`Brewfile` と `brew bundle`）。

---

## インストール先の違い（環境別）

| 環境 | 典型プレフィックス | メモ |
|------|-------------------|------|
| Apple Silicon Mac | `/opt/homebrew` | インストール後に PATH 追記が案内されることが多い |
| Intel Mac | `/usr/local` | 従来からよく使われる場所 |
| Linux（既定） | `/home/linuxbrew/.linuxbrew` | ドキュメント上の Linux 向け既定例 |

実体はプレフィックス配下に置き、実行ファイルなどはシンボリックリンクで PATH に載る構成。

---

## 主要な概念（HOW の前提）

| 用語 | 意味 |
|------|------|
| **Formula（フォーミュラ）** | コマンドラインツールやライブラリなどをビルド／取得するための「レシピ」。`brew install git` の `git` がこれに相当。 |
| **Cask（カスク）** | macOS 向けに、GUI アプリやフォントなどを配布単位で扱う仕組み。`brew install --cask visual-studio-code` のように `--cask` を付ける。 |
| **Tap（タップ）** | Formula を追加で配布するリポジトリ。コア以外の式を `brew tap 所有者/リポジトリ名` で取り込む。 |

---

## 典型的な利用フロー

1. **公式手順でインストール**（インストールスクリプトまたは公式の `.pkg` など、ドキュメント記載の方法）。
2. **PATH を通す**（ターミナルで `brew` が叩けることを確認）。
3. **必要なツールを入れる**（Formula / Cask を使い分け）。
4. **定期的に更新**（`brew update` でカタログ更新 → `brew upgrade` でパッケージ更新、など運用ルールを決める）。
5. **不調時は診断**（`brew doctor` でよくある設定ミスを確認）。

---

## よく使うコマンド（抜粋）

| コマンド | 役割 |
|----------|------|
| `brew install <名前>` | Formula をインストール |
| `brew install --cask <名前>` | Cask をインストール |
| `brew uninstall <名前>` | アンインストール |
| `brew update` | 式の一覧（レジストリ）を更新 |
| `brew upgrade` | インストール済みを新しい版へ更新 |
| `brew list` | インストール一覧 |
| `brew search <文字列>` | 検索 |
| `brew doctor` | 環境診断 |
| `brew cleanup` | 古いバージョン等の掃除 |

---

## 期待効果（RESULT）

- **セットアップ時間の短縮**: 同種のツールを繰り返し入れ替えしやすい。
- **再現性**: `Brewfile` で依存を宣言し `brew bundle` で一括インストール可能。
- **コミュニティ資産**: メンテされた Formula / Cask が公式・サードパーティから提供される。

---

## 注意点・次のアクション（NEXT）

- インストールスクリプトを **curl で取得して bash に渡す**手順は一般的だが、**必ず公式ドキュメント（brew.sh / docs.brew.sh）の URL** を確認してから実行する習慣が望ましい。
- Apple Silicon では **PATH に `/opt/homebrew/bin` が含まれているか**を最初に確認するとトラブルが減る。
- プロジェクト用に `Brewfile` を置き、オンボーディング手順に `brew bundle` を書いておくとチーム共有に便利。
