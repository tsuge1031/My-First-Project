# Claude Code 社内導入提案

**〜 AIエージェントハブとして、全業務を変える 〜**

> 対象：全部門・役員　｜　2026年3月28日　｜　機密区分：社内限り

コードを書くツールではない。あらゆるAIサービス・社内システムと連携し、開発・営業・管理・提案まで横断的に業務を自動化する次世代AI基盤の導入提案。

## 01｜Claude Code とは何か——「AIのハブ」という本質

Claude Code は「コードを書くツール」ではなく、**あらゆるAIサービス・社内システムをつなぎ、自律的に業務を遂行するAIエージェントハブ**です。
その中核技術が **MCP（Model Context Protocol）**——「AIのUSB-C」とも呼ばれる、AIと外部システムをつなぐ業界標準の接続プロトコルです。

### MCPハブ構成図

```
【左側：入力システム】              【右側：出力・実行システム】
📧 メール / カレンダー              🔎 Web検索・情報収集
   Gmail / Outlook    ⇄              Google / Bing
                         🤖
📋 プロジェクト管理               💻 開発ツール
   Jira / Notion      ⇄  Claude     GitHub / CI/CD
                         Code
💬 コミュニケーション   ⇄  (AIエージェントハブ)   💳 決済・会計
   Slack / Teams                   Stripe / 弥生
                         ⇄
🗄️ 社内DB・基幹システム            🌐 ブラウザ操作
   PostgreSQL / SAP               Playwright MCP

              ↑ すべて MCP（Model Context Protocol）で接続 ↑
```

> **「指示するだけで、AIが複数のシステムをまたいで自律的に業務を完遂する」**
> 人間は「何を達成したいか」を伝えるだけ。Claude Code がシステムを横断し、調査・判断・実行・報告まで一気通貫で行う。

## 04｜Microsoft Copilot との比較

| 比較項目 | Microsoft Copilot（現在導入中） | Claude Code（提案） |
|---------|-------------------------------|-------------------|
| **基本コンセプト** | IDE上でのコード補完が中心。エージェントとしての自律実行は限定的 | ✅ **複数システムをまたいで自律的にタスクを完遂するAIエージェントハブ（優位）** |
| **コンテキストウィンドウ** | 約128K tokens（大規模コードベースでは限界あり） | ✅ **200K tokens — 大規模システムの全体把握が可能（優位）** |
| **外部システム連携** | M365アプリ（Teams・SharePoint・Outlook）中心。外部システム連携は限定的 | ✅ **MCPにより社内DB・Slack・Jira・Stripe・GitHub等あらゆるシステムと接続（優位）** |
| **業務自動化の範囲** | Word・Excel・Teams等の操作補助が主。業務自動化の範囲はM365内に限られる | ✅ **開発・営業・管理・会計・採用など全業務領域をシステム横断で自動化（優位）** |
| **コード補完（日常開発）** | ✅ **SWE-benchスコア：約44%（IDE補完に最適化・得意）** | ターミナルベースのため、インライン補完はCopilotが上。用途が異なる |
| **複雑タスクの自律実行** | 複数ファイル横断の大規模変更・フレームワーク移行は苦手 | ✅ **SWE-bench 72.5%達成。40ファイル超の大規模改修も自律実行（大きく優位）** |
| **コスト・導入容易性** | ✅ **M365ライセンスに追加。$30/user/月（既存環境）** | 別途APIコスト必要。ただし全社20名で月約¥75,000（1名あたり¥3,750） |
| **拡張性・カスタマイズ** | Microsoft主導のロードマップに依存。自社独自の連携構築は困難 | ✅ **MCPサーバーを自社開発可能。顧客システムへの独自連携を構築・提供できる（優位）** |

> Microsoftが競合のGPTではなく**Claudeの知性を自社製品に採用した**という事実は、Claude Codeの優位性を端的に示しています。
> つまり「MicrosoftとのリレーションシップとClaude Code導入は矛盾しない」——むしろMicrosoftも認めたAIを先行活用することになります。

## 05｜メリット・デメリット

### ✅ メリット（導入効果）

1. **全業務部門の生産性向上**
   開発だけでなく営業・管理・企画も対象。定型業務・情報収集・文書作成をAIが代替し、人材を高付加価値業務に集中させる。

2. **既存システムとの即時連携**
   MCPにより社内DB・Slack・Jira・基幹システム等と接続。新規開発不要で既存IT資産をAIから活用できる。

3. **顧客への新サービス展開**
   自社導入のノウハウを武器に、AIエージェント導入支援・MCP構築を顧客向けサービスとして提供。新規収益源を創出。

4. **業界標準技術の先行習得**
   MCPはOpenAI・Google等も採用する業界標準。今から習熟することで、AI統合案件における技術的優位性を確立できる。

5. **競争力・差別化の実現**
   AI活用を提案力に変換。「AIを使いこなすSIer」としてのブランドイメージ向上と、工数削減による価格競争力向上を両立。

### ⚠️ デメリット・留意点

1. **顧客情報の外部送信リスク**
   社内DBや顧客データに接続するMCPが強力なため、設定ミスによる機密情報の外部流出リスクが高まる。適切なアクセス制御が必須。→ 対策あり（06参照）

2. **エージェントの暴走リスク**
   自律的に動くため、意図しないファイル削除・メール送信・DB更新が起きる可能性がある。権限設計と人間の承認フロー設計が重要。

3. **MCP設計・管理の専門知識**
   MCPサーバーの構築・管理にはエンジニアリングの知識が必要。導入初期は社内に知見のある担当者の育成が必要。

4. **ランニングコストの変動**
   エージェントが複数システムを呼び出すほどAPI使用量が増加。利用量の上限設定と定期的なコスト監視が必要。

5. **AI出力への過信リスク**
   自律実行の便利さから人間のチェックが薄れる可能性。重要業務は必ずヒューマンインザループ（人間の最終確認）を維持するルール設計が必要。

## 06｜セキュリティ対策（独立系SIerとして最重要）

| 対策 | 内容 |
|------|------|
| 🔒 **学習利用なし** | Team/Enterprise/APIプランでは送信データがモデル学習に一切使用されないことを契約で保証 |
| 🗄️ **ゼロデータ保持（ZDR）** | Enterpriseプランで申請可能。推論後にプロンプト・レスポンスをサーバーから即時削除。官公庁・金融案件にも対応 |
| ☁️ **自社クラウド内完結** | AWS Bedrock / Google Vertex AI経由でデータがAnthropicサーバーに送られず自社環境内で完結 |
| ⚙️ **MCPアクセス権限制御** | MCP接続先ごとに読み取り専用・書き込み禁止などの権限を細かく設定可能。最小権限の原則を実装 |
| 📋 **監査ログ完備** | AWS CloudTrailによる自動記録。AIの全アクションを追跡・証跡化。コンプライアンス対応に活用 |
| 🛡️ **国際認証取得済み** | SOC 2 Type II・ISO 27001・ISO/IEC 42001（AI管理）・HIPAA BAA対応 |

### 推奨導入方式

| 導入方式 | データの流れ | 監査ログ | セキュリティ | 推奨度 |
|---------|------------|---------|------------|-------|
| Anthropic API直接 | Anthropicサーバー経由 | なし | 標準 | PoC・検証用のみ |
| **AWS Bedrock経由** | **自社AWSの範囲内で完結** | **CloudTrail自動記録** | **高い** | **★ 推奨** |
| Enterprise + ZDR | 推論後即時削除 | Compliance API | 最高水準 | 官公庁・金融案件向け |

### 🎯 経営判断のポイント

**1. 「コードを書くAI」ではなく「業務基盤としてのAI」**
Claude Code は開発部門だけのツールではない。MCPによって営業・管理・企画のあらゆる業務システムと接続し、全社の生産性を底上げする基盤投資として捉えるべきである。

**2. シャドーAIを放置することの方がリスクが大きい**
社員が個人でAIツールを無断利用している可能性が高い。公式導入により利用をガバナンスの下に置く方が、顧客情報漏洩リスクを確実に低減できる。

**3. MCP習熟は今後の受注競争力に直結する**
MCPはOpenAI・Googleも採用する業界標準。今から自社に実装することで、顧客のAIシステム統合案件を受注する技術的優位性を早期に確立できる。

**4. 推奨アクション：まずPoC承認を**
AWS Bedrock経由のセキュアな環境で、開発・非開発混合チームによる1〜2ヶ月のPoC実施を承認いただきたい。定量効果の確認後に全社展開・顧客サービス化を判断する。
